# RedisVL Parity Matrix

This document tracks the Rust implementation against the current Python
`redis-vl` surface.

Validation policy: upstream Python tests under `redis-vl-python/tests` are the
source of truth for behavior parity.

| Area | Target | Status | Notes |
| --- | --- | --- | --- |
| Schema | `IndexSchema`, YAML loading, typed field attrs | **Complete** | YAML/JSON parsing, field validation, stopwords, hash/json storage, multi-prefix support, and key-separator normalization are implemented with full unit and integration coverage |
| Filters | Tag/Text/Num/Geo/GeoRadius/Timestamp | **Complete** | Core DSL, boolean composition, and Redis syntax rendering are implemented with comprehensive unit and integration coverage |
| Queries | Vector/Range/Text/Filter/Count/Hybrid/AggregateHybrid/MultiVector | **Complete** | All query types are implemented with command builders. `HybridQuery` generates `FT.HYBRID` commands (requires Redis 8.4+). `AggregateHybridQuery` generates `FT.AGGREGATE` commands. `MultiVectorQuery` generates multi-vector aggregate commands. Redis 8.4 integration tests are implemented and environment-gated in `python_parity_hybrid_aggregate.rs` |
| Search Index | Sync + async lifecycle/load/info | **Complete** | Sync/async create/delete/load/fetch/search/query/batch/paginate, `hybrid_search`/`hybrid_query`, `aggregate_query`, `multi_vector_query`, `sql_query` (auto-dispatch to `FT.SEARCH`/`FT.AGGREGATE`), and `from_existing` are implemented; multi-prefix index creation is supported; Redis 8.4 integration tests are environment-gated |
| SQL | `SQLQuery` behind `sql` feature | **Complete** | SQL→Redis Search translation for non-aggregate `SELECT` queries: `WHERE` (tag/numeric/text/date comparisons, `IN`/`NOT IN`, `LIKE`/`NOT LIKE`, `BETWEEN`, `AND`/`OR`), `ORDER BY`, `LIMIT`/`OFFSET`, field projection. Aggregate SQL (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`, `STDDEV`, `COUNT_DISTINCT`, `QUANTILE`, `GROUP BY`) is translated to `FT.AGGREGATE`; `SearchIndex::sql_query`/`AsyncSearchIndex::sql_query` auto-dispatch between `FT.SEARCH` and `FT.AGGREGATE`. Vector search functions (`vector_distance()`, `cosine_distance()`) translate to KNN `FT.SEARCH` commands. Geo functions (`geo_distance()`) are supported in both WHERE clauses (→ `GEOFILTER`) and SELECT clauses (→ `FT.AGGREGATE` with `APPLY geodistance`). Remaining out of scope: date functions (`YEAR()`), `IS NULL`/`IS NOT NULL`, `HAVING`, phrase-level stopwords. Note: The SQL parser uses a hand-rolled tokenizer rather than `sqlparser-rs`; this is the recommended approach for now as it keeps the dependency footprint small and provides Redis-specific translation control |
| Vectorizers | OpenAI/LiteLLM/Custom + Azure/Cohere/VoyageAI/Mistral/Vertex AI/Bedrock/Anthropic/HF-local | **Complete** | `OpenAITextVectorizer`, `LiteLLMTextVectorizer`, `CustomTextVectorizer`, `AzureOpenAITextVectorizer`, `CohereTextVectorizer`, `VoyageAITextVectorizer`, `MistralAITextVectorizer`, `VertexAITextVectorizer`, `BedrockTextVectorizer`, and `AnthropicTextVectorizer` (Voyage AI-backed adapter — Anthropic recommends Voyage AI for embeddings rather than providing native embeddings) are implemented and wired behind feature flags. `HuggingFaceTextVectorizer` (behind `hf-local` feature, powered by `fastembed` / ONNX Runtime for local inference) is implemented. Live provider tests remain out of CI; feature compilation and request/response unit tests are the current contract |
| Rerankers | `CohereReranker` behind `rerankers` feature | **Complete** | `Reranker`/`AsyncReranker` traits and `CohereReranker` with sync/async support are implemented. Additional reranker providers can be added incrementally |
| Cache Extensions | Semantic + embeddings cache | **Complete** | `EmbeddingsCache` and `SemanticCache` are Redis-backed and parity-tested with comprehensive sync/async coverage (12 embeddings-cache tests, 19 semantic-cache tests). `SemanticCache` also provides `with_default_vectorizer()` behind the `hf-local` feature for zero-config local embedding |
| Message History | MessageHistory | **Complete** | Standard message history is Redis-backed and parity-tested for ordered retrieval, scope, drop/count, and role filtering (4 history tests + 3 role-filter tests) |
| Semantic Message History | SemanticMessageHistory | **Complete** | Semantic message history is Redis-backed and parity-tested for ordered retrieval, semantic recall, scope, drop/count, and role filtering (7 integration tests). Dtype selection, overwrite control, and reconnect with schema mismatch detection are implemented. `with_default_vectorizer()` convenience method is available behind the `hf-local` feature |
| Router | SemanticRouter/Route/RouteMatch | **Complete** | Redis-backed semantic router is implemented and parity-tested for routing, updates, and lifecycle behavior (20 integration tests). Dtype selection, overwrite control, reconnect with schema mismatch detection, `from_existing`, `from_dict`/`from_yaml` serialization, and route reference management are implemented. `with_default_vectorizer()` convenience method is available behind the `hf-local` feature |
| CLI | `rvl version/index/stats` | In progress | Command surface includes `version` (with `--short`), `index create/delete/destroy/info/listall`, and `stats` (Python-matching `STATS_KEYS`). Supports `--index` name or `--schema` file for index identification, `--redis-url`/`REDIS_URL` for connection, and `--delete-documents` for destructive deletes (21 CLI smoke tests). Further CLI parity work remains for `load`, query, and richer output formatting |
| Benchmarks | Criterion micro-benchmarks + Redis-backed benchmarks | In progress | `criterion` benchmarks for schema parsing, filter rendering, and query building exist. Redis-backed benchmarks cover search index operations (create/exists/info/load/fetch), search queries (vector/filter/count/batch/paginate), embeddings cache (set/get hit/miss), semantic cache (store/check hit/miss), message history (add/get_recent), and semantic message history (add/get_recent/get_relevant). Rust-vs-Python comparison harness is not yet implemented |
| Docs/Examples | README/examples/API docs | In progress | Repo scaffolding, mdBook guide, and rustdoc exist but example coverage and publication-quality docs are still incomplete |
