# API Reference

The full Rust API documentation is generated by `cargo doc` and published
alongside this guide.

## Browsing the API docs

If you are viewing this on the GitHub Pages site, the API docs are available
at [`./api/redis_vl/`](api/redis_vl/index.html).

You can also generate them locally:

```bash
cargo doc --workspace --all-features --no-deps --open
```

## Key types at a glance

| Type | Module | Description |
| --- | --- | --- |
| `IndexSchema` | `schema` | Index definition with fields, prefixes, and storage config |
| `SearchIndex` | `index` | Sync search index lifecycle and query execution |
| `AsyncSearchIndex` | `index` | Async search index lifecycle and query execution |
| `VectorQuery` | `query` | K-nearest neighbor vector search |
| `VectorRangeQuery` | `query` | Vector search within a distance threshold |
| `TextQuery` | `query` | Full-text search query |
| `FilterQuery` | `query` | Filter-only search |
| `CountQuery` | `query` | Document count query |
| `HybridQuery` | `query` | Combined text + vector search (Redis 8.4+) |
| `SQLQuery` | `query` | SQL → Redis Search translation (`sql` feature) |
| `FilterExpression` | `filter` | Composable filter DSL |
| `Tag` / `Text` / `Num` / `Geo` / `Timestamp` | `filter` | Filter builders |
| `Vectorizer` / `AsyncVectorizer` | `vectorizers` | Embedding provider traits |
| `Reranker` / `AsyncReranker` | `rerankers` | Reranking provider traits |
| `SemanticCache` | `extensions::cache` | LLM response caching with semantic similarity |
| `EmbeddingsCache` | `extensions::cache` | Deterministic embedding vector cache |
| `MessageHistory` | `extensions::history` | Conversation message storage |
| `SemanticMessageHistory` | `extensions::history` | Semantic recall message history |
| `SemanticRouter` | `extensions::router` | Vector-based text classification |
